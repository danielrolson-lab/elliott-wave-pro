# Elliott Wave Pro — Launch Checklist

Pre-submission checklist for App Store (iOS) and Google Play (Android).
Check off each item before submitting for review.

---

## 1. Infrastructure

### Supabase
- [ ] Run `supabase/migrations/001_initial_schema.sql` against production project
- [ ] Verify RLS is active on all tables (`select * from pg_policies`)
- [ ] Confirm `on_auth_user_created` trigger fires correctly (test signup)
- [ ] Enable Supabase Realtime for `wave_count_cache` and `market_regimes` tables
- [ ] Set `SUPABASE_SERVICE_ROLE_KEY` in Vercel and Fly.io environments

### Vercel (Proxy)
- [ ] Deploy `services/proxy/` with `vercel --prod`
- [ ] Set env vars: `ANTHROPIC_API_KEY`, `SUPABASE_URL`, `SUPABASE_SERVICE_ROLE_KEY`, `UPSTASH_REDIS_REST_URL`, `UPSTASH_REDIS_REST_TOKEN`
- [ ] Smoke test: `POST /api/ai-commentary` returns 200
- [ ] Smoke test: `POST /api/alert-intelligence` returns 200
- [ ] Smoke test: `GET /api/wave-count?ticker=SPY` returns 200
- [ ] Confirm `ANTHROPIC_API_KEY` is NOT in `EXPO_PUBLIC_` variables

### Fly.io — Wave Scanner (FastAPI)
- [ ] `cd services/fastapi && fly deploy`
- [ ] `fly status` — app is running
- [ ] Test: `curl -X POST https://elliott-wave-scanner.fly.dev/wave-scan -d '{"ticker":"SPY","timeframe":"5m","lookback_days":30,"wave_type":"impulse"}'`
- [ ] Set `POLYGON_API_KEY` secret: `fly secrets set POLYGON_API_KEY=...`

### Fly.io — Wave Stream (WebSocket)
- [ ] `cd services/proxy && fly deploy`
- [ ] `wscat -c wss://elliott-wave-stream.fly.dev` — connection accepted
- [ ] Auth message `{"type":"auth","api_key":"test"}` returns correct error
- [ ] Set Supabase secrets: `fly secrets set SUPABASE_URL=... SUPABASE_SERVICE_ROLE_KEY=...`

### Upstash Redis
- [ ] Create Redis database at upstash.com
- [ ] Copy REST URL and token to Vercel env vars
- [ ] Confirm 30-second signal cache is working (check hit ratio after first requests)
